# 龙虾军团 (Lobster Legion) - 演进与开发计划

基于“龙虾军团”（OpenClaws 管理平台）的 MVP 闭环逻辑，以及你“**不想引入复杂外部中间件、充分利用 OpenClaw 原生扩展能力**”的指导思想，我为你量身定制了这份高度可行的落地开发计划。

我们将摒弃 MQTT/Redis 等沉重的设计，转而利用 OpenClaw 极其强大的 **Channel (频道) 插件机制**，在我们的 Electron 平台内部闭环实现“群聊”和“协同”功能。

---

## 阶段一：本地“议事厅”基建与扩展接入 (核心基础)
*目标：不依赖飞书/Slack，纯本地实现一个仿飞书的通信频道，让主/从龙虾都能连进来。*

1. **[平台端] 在 Koa 主进程中实现简易 Chat Server**
   - 在 Electron 后端 (Koa) 中，暴露一个精简的 HTTP / WebSocket 接口（类似于我们自己做了一个微型飞书服务器）。
   - 提供发送消息、拉取历史消息、接收 Webhook 的基础端点。
   - 数据暂存使用我们现有的 JSONL 文件系统（`chat_history.jsonl`）。

2. **[OpenClaw 端] 编写自定义扩展插件 (`extensions/lobster-chat`)**
   - 在 OpenClaw 源码的 `extensions/` 目录下，新建一个专门用于对接我们本地服务器的 Channel Plugin。
   - 该插件参考官方 `feishu` 频道的实现，将接收到的 OpenClaw 消息（`api.sendMessage`）通过 HTTP/WS 转发到我们的 Electron Koa 服务器。
   - 该插件能持续监听 (Poll/WS) 我们的 Koa 服务器，收到用户发言后触发 OpenClaw 的内部处理逻辑。

3. **[容器侧] 在容器构建与配置中挂载/启用新插件**
   - 调整 `dockerManager` 和 `openclaw.json` 生成逻辑，确保每个拉起的龙虾容器都会加载并激活这个我们手写的 `lobster-chat` 扩展。
   - 将 Koa 服务器的内网 IP (`host.docker.internal:18888`) 作为配置注入到 `openclaw.json` 中。

---

## 阶段二：打通“龙虾议事厅”用户界面 (人机交互)
*目标：在前端有一个群组聊天界面，你能在这个界面里直接看到所有龙虾的聊天和工作汇报。*

4. **[前端产品] 设计并开发“龙虾议事厅”视图**
   - 在左侧菜单栏增加“议事厅” (Lobby/Hub) 模块。
   - 仿照群聊界面：左侧显示不同的话题/群组，右侧是聊天流。
   - 聊天流中，发言人可以是：你（人类）、Master 龙虾、Worker 龙虾。

5. **[前端联调] 前后端消息的实时双向绑定**
   - 前端 Vue3 组件通过 WebSocket 连接到我们第一步写的 Koa Chat Server。
   - 你在前端发一句话（如：“大家现在进度怎么样了？”），Koa 会将其广播。
   - 各个龙虾内部的 `lobster-chat` 插件收到广播，处理后回复消息，展示在群聊中。

---

## 阶段三：主从协同流与任务看板 (业务闭环)
*目标：基于“群聊通道”与“Gitea”，让业务流转清晰可见。*

6. **[业务联调] Gitea Webhook 接入“议事厅”**
   - 在 Koa 服务中增加接收 Gitea 仓库 Webhook 的端点。
   - 当仓库中有 Issue 被创建或 PR 提交时，Koa 服务将其格式化为一条群消息，发送到“议事厅”的群聊里（例如：【系统广播】"新的任务 #1 已分配"）。
   - Master 龙虾看到该群消息后，自动认领并触发它的思维链（思考如何拆解任务）。

7. **[功能跑通] 修复并集成“任务大厅 (Tasks)”模块**
   - 解决前端现有 Tasks 页面的问题，直接对接我们自己保存的或 Gitea 的工单数据。
   - 实现任务从“待办 -> 处理中 -> Code Review -> 完成”的可视化流转。

8. **[工作流模板] 基于角色的快速开箱模板**
   - 在“新建龙虾”界面，将“主/从”的选项具象化。提供自带 `SOUL.md` 和预设工具白名单的模版，如：
     - **技术经理 (Master)**：专注于读 Issue，分拆任务。
     - **打工仔 (Worker)**：专注于写代码、跑测试、提 PR。
     - **质量检测员 (QA)**：专注于在群里审查别人提的代码。

---

## 阶段四：易用性打磨与功能深化
*目标：提升细节体验，确保系统好用且不崩。*

9. **[资产管理] 龙虾记忆 (Memory) 的可视化管理**
   - 在龙虾详情页增加一个“记忆管理”Tab，展示它的核心上下文文件（如它对架构的理解），允许你手动微调纠正。

10. **[资源与模型] 支持单个龙虾无缝切换模型与代理**
    - 在现有的动态模型加载基础上，完善容错机制。在界面上更清晰地展示当前龙虾实际请求的底层供应商。
    - 添加针对不同角色分配不同模型的建议（比如用 Qwen-Max 做 Master 拆解任务，用 Qwen-Coder 做 Worker 写代码）。
