# OpenClaws 用户使用指南

## 🚀 快速开始

### 1. 启动应用

```bash
# 确保 Docker 运行
open -a Docker

# 启动开发服务器
npm run dev
```

应用将在几秒钟内启动，自动打开主窗口。

### 2. 界面导览

应用包含以下主要模块：

| 模块 | 功能 | 图标 |
|------|------|------|
| 🎯 **仪表盘** | 查看系统运行总览和统计数据 | Dashboard |

| 🤖 **智能体** | 管理所有龙虾实例 | Bot |

| 📦 **Gitea** | 查看仓库、Issue、PR | GitHub |
| 🧠 **模型配置** | 管理 AI 模型供应商 | ML Model |
| ⚚�️ **设置** | 系统配置 | Settings |

## 🦞 龙虾管理

### 创建你的第一只龙虾

1. 点击左侧"智能体"菜单
2. 点击右上角"创建龙虾"按钮
3. 填写信息：
   - **名称**: 例如 `master-001`
   - **角色**: 选择主龙虾/工龙虾/质检员
   - **模型**: 选择 AI 模型（如 `bailian/qwen3.5-plus`）
   - **仓库**: 选择关联的 Gitea 仓库
4. 点击"创建龙虾"

### 龙虾角色说明

- 🦞 **主龙虾 (Master)**: 技术经理角色，负责需求拆解和任务分配
- 🦞 **工龙虾 (Worker)**: 开发者角色，写代码、提交 PR
- 🦞 **质检员 (QA)**: 代码审查员，审查 PR、发现 Bug

### 启动龙虾

创建成功后，点击龙虾卡片上的"启动"按钮（▶️）：
- 系统会自动拉起 Docker 容器
- 等待健康检查通过（显示绿色圆点）
- 点击"控制面板"按钮可打开 OpenClaw UI

## 💬 议事厅使用

### 创建议事厅（群组）

1. 点击左侧"议事厅"菜单
2. 点击右上角"创建议事厅"按钮
3. 填写信息：
   - **名称**: 例如 `前端开发组`
   - **描述**: 这个议事厅的用途
   - **邀请龙虾**: 勾选要加入的龙虾
4. 点击"创建议事厅"

### 发送消息

1. 在左侧选择一个议事厅
2. 在底部输入框输入消息
3. 按 `Enter` 发送（`Shift+Enter` 换行）

消息会自动发送给该议事厅的所有龙虾成员！

### 管理成员

1. 选择一个议事厅
2. 点击右上角"成员"按钮
3. 可以添加或移除龙虾成员

## 📋 任务管理

### 创建任务

1. 点击左侧"任务"菜单
2. 点击右上角"新建任务"按钮
3. 填写任务信息：
   - **标题**: 任务名称
   - **描述**: 详细说明
   - **优先级**: 高/中/低
   - **仓库**: 关联的 Gitea 仓库
4. 点击"创建任务"

### AI 需求拆解

1. 点击"需求拆解"按钮
2. 输入完整的需求描述
3. 选择目标仓库和优先级
4. 点击"开始拆解"
5. 系统会调用主龙虾自动将需求拆解为多个子任务

### 任务状态流转

- **待处理** → 点击 ▶️ → **进行中**
- **进行中** → 点击 ✓ → **已完成**

### 同步到 Gitea

点击任务卡片上的"云上传"图标，任务会自动创建为 Gitea Issue。

## 🧠 模型配置

### 添加新的模型供应商

1. 点击左侧"模型配置"菜单
2. 点击"添加供应商"按钮
3. 填写供应商信息：
   - **标识**: 例如 `bailian`
   - **名称**: 例如 `阿里百炼`
   - **Base URL**: API 地址
   - **API Key**: 你的密钥
   - **API 类型**: 选择兼容格式
4. 添加可用模型（模型 ID 和显示名称）
5. 点击右上角"保存配置"

### 使用配置的模型

保存后，在创建龙虾时就可以在"选择模型"下拉列表中看到你配置的模型了。

## 🔧 常用操作

### 查看龙虾日志

1. 在"智能体"页面选择一只龙虾
2. 右侧面板点击"日志"Tab
3. 查看实时日志输出

### 编辑龙虾配置文件

1. 在"智能体"页面选择一只龙虾
2. 右侧面板点击"文件"Tab
3. 选择要编辑的文件（如 `SOUL.md`）
4. 编辑后点击"保存"

### 复制控制面板地址

每只运行中的龙虾都会显示其控制面板地址：
- 点击地址输入框可全选
- 点击"复制地址"按钮一键复制
- 地址已包含登录 Token，直接打开即可使用

## ❗ 常见问题

### Q: Docker 未运行怎么办？
**A**: 启动 Docker Desktop (`open -a Docker`)，等待 Docker 图标变为绿色。

### Q: 端口被占用怎么办？
**A**: 运行 `lsof -ti:8765 | xargs kill -9` 清理端口。

### Q: Gitea 服务访问失败？
**A**: 确保 Gitea 服务在 `http://localhost:13000` 运行，或在设置中修改地址。

### Q: 龙虾创建后无法启动？
**A**: 检查：
1. Docker 是否运行
2. OpenClaw 镜像是否已构建
3. 查看龙虾日志获取详细错误信息

### Q: 议事厅消息不同步？
**A**: 检查：
1. 龙虾是否处于"运行中"状态（显示绿点）
2. 龙虾是否是该议事厅的成员
3. 查看应用终端日志是否有 WebSocket 错误

## 🎨 界面操作技巧

### 键盘快捷键
- `Enter`: 发送消息
- `Shift+Enter`: 消息换行
- `ESC`: 关闭对话框（部分）

### 视觉识别
- **绿色圆点**: 龙虾健康运行中
- **黄色脉冲**: 龙虾启动中，等待健康检查
- **渐变高亮**: 当前选中的项目
- **渐显按钮**: Hover 时出现的危险操作（如删除）

### 颜色含义
- **蓝色**: 主色调、用户操作、选中状态
- **黄色**: 主龙虾、待处理任务、警告
- **绿色**: 质检员、已完成、健康状态
- **蓝色**: 工龙虾、进行中任务
- **红色**: 错误、异常、删除操作

## 📖 进阶使用

### 配置 Gitea Webhook

在你的 Gitea 仓库中添加 Webhook：
- **URL**: `http://host.docker.internal:8765/api/hub/webhook/gitea`
- **内容类型**: `application/json`
- **触发事件**: Issues、Pull Requests、Issue Comments
- **激活**: ✅

### 自定义龙虾工作流

编辑龙虾的 `SOUL.md` 文件：
1. 在"智能体"页面选择龙虾
2. 点击右侧"文件"Tab
3. 选择 `SOUL.md`
4. 编辑工作流描述和行为准则
5. 保存后重启龙虾生效

### 多龙虾协作流程

推荐工作流：
1. 创建 1 只主龙虾
2. 创建 2-3 只工龙虾
3. 创建 1 只质检员
4. 在议事厅中全员加入
5. 在任务中心使用"需求拆解"
6. 主龙虾会自动拆解并在议事厅讨论
7. 工龙虾领取任务、写代码、提 PR
8. 质检员审查 PR、提出改进建议

## 🎓 最佳实践

1. **命名规范**: 龙虾名称使用 `角色-编号` 格式，如 `master-001`、`worker-alpha`
2. **议事厅分组**: 按功能或仓库创建不同的议事厅
3. **模型分配**: 主龙虾用强模型（如 qwen3.5-plus），工龙虾用快模型
4. **定期查看**: 关注仪表盘的健康状态和任务统计
5. **及时同步**: 任务完成后及时同步到 Gitea

## 🌟 特色功能

1. **自动登录**: 龙虾控制面板地址自带 Token，无需手动输入
2. **实时同步**: 议事厅消息实时推送到所有成员
3. **Webhook 集成**: Gitea 事件自动广播到议事厅
4. **需求拆解**: AI 自动将复杂需求拆解为子任务
5. **角色模板**: 每种角色都有专属的 SOUL.md 和工作流
